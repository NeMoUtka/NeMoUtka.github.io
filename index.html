Research Motivation

Why This Matters

Defect detection sits at the intersection of computer vision and industrial automation — yet practical adoption in small manufacturing settings remains limited.

The Adoption Gap

Despite rapid progress in Machine Learning and AI, many small manufacturing companies still avoid these technologies. The primary barriers are perceived cost and technical complexity — not a lack of need. This creates a significant gap between what is technically possible and what is actually deployed on the factory floor.

The Cost of Manual Inspection

Manual defect detection is both expensive and time-consuming. It demands additional employees, dedicated inspection time, and manual result tracking — resources that compound in cost as production scales. Automated inspection directly addresses these inefficiencies while improving consistency and repeatability.

Model Selection

Four Models, Two Approaches

Two learning paradigms — supervised and unsupervised — were each tested with two backbone architectures: a CNN-based ResNet50 and a Transformer-based Swin-Tiny. All models were evaluated on the same test set using identical metrics to ensure a fair comparison.

Supervised

ResNet50

CNN Backbone

A classic convolutional architecture trained on labeled defect annotations, learning discriminative features through direct supervision.

Supervised

Swin-Tiny

Transformer Backbone

A hierarchical vision Transformer fine-tuned on annotated defect data, capturing long-range spatial dependencies across image patches.

Unsupervised

No-Teacher ResNet50

CNN Backbone

An anomaly detection model trained only on normal samples. Defects are identified as deviations from the learned normal distribution.

Unsupervised

No-Teacher Swin-Tiny

Transformer Backbone

A Transformer-based anomaly detector using the same teacher-free paradigm, leveraging self-attention to model normal appearance.

Dataset

MVTec Anomaly Detection

Two product categories from the MVTec AD benchmark were selected — Cable and Transistor — chosen to represent different surface textures and defect types. The same underlying data was adapted differently for each learning paradigm.

Supervised Setup

The dataset was adapted for supervised training using the available defect annotations. Online augmentation — including random flips, crops, and color jitter — improved sample diversity without duplicating images, helping the model generalise across visual variations.

Unsupervised Setup

For the no-teacher models, only defect-free samples were used during training. The models learn a representation of normal appearance, then flag any image region that deviates significantly from that baseline as a potential defect at inference time.

Future Work

Next Steps

Several directions remain open for extending this research toward real-world production deployment.

📷 Smart-Camera Deployment

The primary goal is to deploy trained models on smart cameras for real-time production-line inspection. Models would run fully locally — flagging defects without sending any data to a remote server — enabling low-latency, privacy-preserving quality control.

🗺 Defect Localisation

Beyond classification, future iterations would highlight the exact location of each defect using pixel-level segmentation masks. This gives operators precise spatial feedback and enables downstream automation of rejection or rework decisions.

🔬 Advanced Anomaly Models

Testing more expressive generative models — such as Variational Autoencoders (VAEs) — is planned to improve unsupervised segmentation quality, particularly for subtle or low-contrast defects that challenge current feature-distance approaches.
